Connection string
This is the core one. It gives you the Postgres URL to put in your app’s env vars, usually DATABASE_URL. Your app, ORM, or server code uses that to connect to Neon.

Neon init
This is the guided setup command, npx neonctl@latest init. Neon says it can configure the Neon MCP server for compatible tools and set up your local development environment in one go. Good if you want Claude Code or your editor to understand Neon more deeply.

IDE extension
This is Neon’s editor integration for VS Code/Cursor. It can detect connection strings, browse schemas, run queries, manage branches, and help configure MCP from inside the editor. Useful if you want a GUI in the IDE.

MCP server
This is the AI-tool bridge. It lets tools like Claude Code interact with Neon in natural language: list projects, create branches, run SQL, and help with schema changes. Neon documents it as intended for development/testing, not production.
